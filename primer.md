# Primer de workflow

But : résumer en quelques lignes les règles de collaboration et de mémoire pour l'assistant.

- Toujours vérifier le dépôt courant: `git status`, `git remote -v`
- Prioriser le dossier exact (ex : `dash`) si la racine n'est pas un git
- Éviter exposition token dans URL remote
- Rester concis pour l'usage de tokens
- Documenter chaque action dans un fichier de suivi (Claude, Obsidian, Hindsight)

## Format de résultat
- Titres suivis de bullet points
- Contexte + actions + décisions
- Liste des corrections urgentes
